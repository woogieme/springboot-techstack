내가 수년간 운영중인 프로젝트에 투입됐다고 가정해보자
거기서 내가 맡은역할은 로그추적기를 다는것.
로그 추적기란?
어플리케이션이 개발이되면 시간이지나면서 모니터링과 운영이 중요해지는 단계가 된다
또한 프로젝트가 점점 병목이 자꾸 생겨서 장애가 나고 그럴때, 어느 부분에서 에러가 생기는지 궁금할때
로그를 미리미리 남겨둔다면 어떨까? 그 역할이 하는것이 로그 추적기
즉, 로그를 개선하고 자동화하는게 로그추적기의 역할 !

정상요청, 예외 발생을  각각 level별로 확인도 가능하고 오류 발생 지점도 확인하고, 걸린 시간까지 볼수있어 좋음

요구사항 -
- 모든 퍼블릭 메소드의 호출과 응답을 로그로 남긴다
- 어플리케이션의 흐름을 변경하면 안됨 (ex. 로그가 남긴다고해서 비지니스 로직에 영향을 주면안됨)
- 메서드 호출에 걸린 시간
- 정상흐름과 예외 흐름을 구분해야한다.
- 메서드 호출의 깊이 표현
- HTTP 요청을 구분
    - HTTP 요청 단위로 특정 ID를 남겨서 어떤 HTTP 요청에서 시작된 것인지 명확하게 구분이 가능해야 함
    - 트랜잭션 ID(DB 트랜잭션 이랑은 다른말임), 여기서는 하나의 HTTP 요청이 시작해서 끝날 때 까지를 하나의 트랜잭션이라고 함
